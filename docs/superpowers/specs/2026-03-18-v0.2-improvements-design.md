# opencode-bell v0.2 Improvements Design

## Context

opencode-bell v0.1.1 is a working, zero-dependency terminal bell plugin for OpenCode CLI. The project is positioned as a learning vehicle for open-source operations practices. Current state: single-file implementation, one regression test, minimal README.

## Goals

1. Learn open-source operations: README craft, GIF demos, community outreach
2. Keep the plugin minimal — terminal bell only, zero dependencies
3. Make the project presentable enough to promote in the OpenCode community

## Non-Goals

- Expanding into a multi-backend notification system
- Adding external dependencies
- Pursuing download/star metrics as a primary objective

## Part 1: Configuration System

### Current State

All behavior is hardcoded: 4 event types, 1200ms debounce.

### Prerequisite

OpenCode's plugin config schema (`z.string().array()`) only accepts string arrays — no tuple/object syntax. The factory function receives `PluginInput` (client, project, directory, etc.) with no user config field. Therefore, plugin-level config must use environment variables.

### Design

Configure via environment variables:

```bash
# Only bell on permission and idle events (comma-separated)
export OPENCODE_BELL_EVENTS="permission.asked,session.idle"

# Custom debounce window in milliseconds
export OPENCODE_BELL_DEBOUNCE="2000"
```

**Variables:**

| Variable | Default | Description |
|----------|---------|-------------|
| `OPENCODE_BELL_EVENTS` | `permission.asked,question.asked,session.idle,session.error` | Comma-separated event types to trigger bell |
| `OPENCODE_BELL_DEBOUNCE` | `1200` | Debounce window in ms per event+session key |

**Implementation:**

- Factory function signature stays `OpencodeBellPlugin()` (no change to contract)
- On init, read `process.env.OPENCODE_BELL_EVENTS` (split by comma, trim) and `process.env.OPENCODE_BELL_DEBOUNCE` (`parseInt(value, 10)`, fall back to 1200 if NaN/negative/zero)
- Filter incoming events against the configured event list; unknown event types are silently ignored
- Debounce key format: `${event.type}:${sessionId}` when sessionId exists, `${event.type}` when missing (no trailing colon)
- No env vars set = identical behavior to v0.1.1 (zero breaking change)
- Internal `ring(now)` helper: `now` is a `number` (timestamp), defaults to `Date.now()` at call site

## Part 5: Test Coverage

### New Test Cases

**Environment variable config:**
- `OPENCODE_BELL_EVENTS` filters events correctly
- `OPENCODE_BELL_DEBOUNCE` overrides debounce window
- `OPENCODE_BELL_DEBOUNCE=abc` (invalid) falls back to default 1200
- No env vars = default behavior unchanged

**Debounce logic:**
- Same key within window: no repeat bell
- Same key after window expires: bell fires again (uses injectable `now` param for deterministic testing)

**TTY guard:**
- Temporarily stub `process.stdout.isTTY = true` then verify `process.stdout.write` is called; stub `= false` then verify it is not called. Restore original value after each test.

### Scope

Core paths only. No coverage target.

- Read `OPENCODE_BELL_EVENTS` and `OPENCODE_BELL_DEBOUNCE` from `process.env` at factory init
- Parse events: split by comma, trim, fall back to all 4 defaults
- Parse debounce: `parseInt(value, 10)`, fall back to 1200 if NaN/negative/zero

- Extract `ring(now)` helper (`now` is a number, defaults to `Date.now()` at call site)
- Filter `event.type` against configured events list
- Commit
